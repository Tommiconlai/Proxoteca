## Overall Impression

A genuinely well-crafted, distinctive tool that has clearly been through several polish rounds. The core flow (add → arrange → export) works, the design identity is committed, and prevention/undo are above average. What's left is concentrated in three places: **(1) keyboard accessibility** — no overlay closes on Escape and focus is never managed; **(2) the export black box** — the single most important action gives a spinner and nothing else, with no way to cancel; **(3) CMYK copy** — the print-shop path is powerful but explains its jargon poorly exactly where the user must decide. The single biggest opportunity is the cheapest: one shared Escape/focus pattern fixes a P1 across every modal, sheet, and tooltip at once.

## What's Working

1. **Committed, non-template identity** — dark OLED + warm gold + grain + light-table preview is a real point of view, executed consistently. Detector-clean.
2. **Prevention + recovery depth** — delete-all confirm, undo-with-toast on delete, `toNum` guards against NaN grids, degenerate-sheet handling, ICC DeviceLink rejection, low-res warnings, beforeunload guard. Rare in a hobby tool.
3. **Power-user flexibility** — desktop multi-select with Ctrl/Shift-range, bulk bleed/delete, deck-link import, printing-pinning, settings persistence, keyboard Space-select. Real accelerators.

## Priority Issues

- **[P1] No Escape / focus management on any overlay** (consolidates 6 findings)
  - **Why:** Keyboard and screen-reader users cannot dismiss ScryfallImportModal, ArtPickerModal, MpcImportModal, ConfirmDialog, the help tooltip, the mobile add-sheet, CardActionSheet, or the Settings/Export MobilePage with Escape. Focus is never moved into a modal on open or restored to the trigger on close. `aria-modal="true"` is set but nothing traps Tab. The pattern already exists in the codebase (PagePreview multi-select Esc) — it just isn't applied to dialogs.
  - **Fix:** One shared `useModalDismiss(onClose)` hook (Escape listener + focus-into-first + restore-on-unmount), applied to every overlay. Convert the help tooltip to `role="dialog"` + Esc, or keep `tooltip` and drop the `aria-haspopup="dialog"` mismatch.
  - **Evidence:** ScryfallImportModal.jsx:57, ArtPickerModal.jsx:31, MpcImportModal.jsx:47, ConfirmDialog.jsx:11, App.jsx:128–133 (mousedown only), MobileLayout.jsx:21/76, CardActionSheet.jsx:10. Only Escape handler in the app is PagePreview.jsx:307 (clears selection).
  - **Suggested command:** `/impeccable harden`

- **[P1] Generate PDF is a no-progress, no-cancel block**
  - **Why:** The headline action shows a spinner + "Generating…" and disables the button — but for 100+ cards at high DPI the synchronous render loop can freeze the UI for tens of seconds with zero progress and no abort. The Scryfall/MPC imports right next to it both show `done/total` progress; the export, which matters most, shows less.
  - **Fix:** Thread a progress callback (cards rendered / total) into `generatePDF`/`generatePDFCmyk` and surface a determinate bar; add an AbortController + Cancel button. Yield to the event loop between cards so the bar paints.
  - **Evidence:** App.jsx:316–341 (no abort/progress), pdfGenerator.js / pdfGeneratorCmyk.js (synchronous per-card loops), vs MpcImportModal.jsx:10–36 / ScryfallImportModal.jsx:40–53 (progress callbacks).
  - **Suggested command:** `/impeccable harden`

- **[P1] The CMYK path under-explains at the decision point**
  - **Why:** The non-actionable error `Load your ICC profile, or pick a bundled one.` (App.jsx:324) names the problem but not where to fix it. Around it, "Relative Colorimetric + BPC" never decodes BPC, the two rendering intents aren't differentiated and the default isn't marked, the ICC upload hint doesn't say where to get a profile or that bundled FOGRA options exist, and the profile-mismatch warning is jargon ("embedded profile… passed through unconverted") that even contradicts the nearby "already in your profile" hint. A print-shop newcomer can't make these calls.
  - **Fix:** Make the ICC error point to the control ("Open CMYK settings → pick Coated FOGRA39 or upload your shop's profile"). Gloss BPC inline, mark the default intent, explain each intent in one line, expand the upload hint, and rewrite the mismatch warning into plain "what happened / should you worry / what to do".
  - **Evidence:** App.jsx:324; PageSettings.jsx:186, 199, 198–202, 189–196, 223–239.
  - **Suggested command:** `/impeccable clarify`

- **[P2] Per-card bleed label lies under a non-auto global style**
  - **Why:** The per-card bleed button shows `bleedLabel(img.bleedMode)` (the per-card mode), but the preview and PDF both run it through `resolveBleedMode(mode, globalStyle)`. Set a card to "stretch" while global Bleed style is "mirror" → the button says one thing, the print does another. In a print-fidelity tool, the label silently misrepresenting output erodes trust.
  - **Fix:** Show the resolved bleed in the label/aria-label, or disable the per-card cycle when global style ≠ auto with a one-line "overridden by Bleed style" hint.
  - **Evidence:** App.jsx:220–223, PagePreview.jsx:435–436 (label) vs :124 (render), pdfGenerator.js:174–184.
  - **Suggested command:** `/impeccable clarify`

- **[P2] A fully-failed import reads as success**
  - **Why:** When every card is not-found, `setResult({imported:0, notFound:[...]})` still renders "✓ 0 images imported" — a green checkmark on total failure — and the not-found list joins unbounded with no scroll/cap, overflowing the modal on a big paste.
  - **Fix:** If `files.length===0 && notFound.length>0`, show an error ("None of N cards were found — check spelling or (SET) num"), not a result box. Cap the not-found list (first 5 + "…and N more") in a scrollable container.
  - **Evidence:** ScryfallImportModal.jsx:43–48, 113–122; index.css .import-notfound (no max-height).
  - **Suggested command:** `/impeccable clarify`

## Persona Red Flags

**Sam (Accessibility-dependent, keyboard + screen reader):** Opens "Import from Scryfall" → focus stays on the trigger behind the overlay (no focus-into-modal). Presses Escape to leave → nothing happens (no Esc on any modal/sheet/tooltip). Tabs → focus walks out of the modal into the page behind it (no trap). On the "Delete all" confirm, focus lands on the destructive button (ConfirmDialog.jsx:19 autoFocus), one Enter from irreversible loss. Reaches CMYK → can't tell "Relative Colorimetric + BPC" from "Perceptual" by listening.

**Jordan (First-timer):** Switches Output to CMYK out of curiosity → hit with PDF/X-1a, DeviceCMYK, embedded ICC, rendering intent, BPC, FOGRA, "passed through unconverted" — none glossed, the help tooltip never mentions CMYK. Tries Generate without a profile → "pick a bundled one" with no clue where. Doesn't discover the keyboard Delete shortcut (undocumented) or that hovering a card reveals duplicate/bleed/delete.

**Casey (One-handed mobile):** Taps `?` on a ≤360px phone → the 320px tooltip drops over the card preview with no visible close button; closing means re-tapping `?` or guessing at tap-outside. The bottom cluster is three icon-only circles (delete / add / save) distinguished only by color — a fast thumb can hit delete instead of add.

## Minor Observations

- **Duplicate-spam → delete leaks object URLs:** `removeWithUndo` schedules a 5.3s revoke timer that isn't tracked; closing the tab inside the window leaks the URLs (App.jsx:201, 96–98). Track pending revokes, or revoke-on-remove and rebuild from `it.file` on undo.
- **Sidebar group headers aren't sticky** (index.css:241–247) — scrolling deep into Print loses the "which group am I in" anchor.
- **Toasts differentiate success/error by border color only** (Toast.jsx) — add a ✓ / ! glyph for instant peripheral read.
- **mm↔inch repeated toggling drifts** sub-mm (PageSettings.jsx:64 rounds 1–2 dp) — store canonical mm, display inches derived.
- **ConfirmDialog destructive autofocus** — move `autoFocus` to Cancel.

## Questions to Consider

- The export is the product's whole point — what would it feel like if Generate showed "card 34 / 96" and an honest Cancel, instead of a spinner you can't trust or stop?
- CMYK is genuinely powerful but speaks like a prepress manual. Who is it for — and could a one-line plain-language gloss per field make it usable by the hobbyist who's the rest of the app's audience?
- Every overlay ignores Escape. If one shared hook fixed Esc + focus everywhere, is there any reason not to?
